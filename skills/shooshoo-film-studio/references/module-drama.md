# 模組二｜劇情式

**製片定調：** 類型由每次企劃決定（懸疑/愛情/職場爽劇/溫情等），世界觀預設以咻咻打包倉庫為核心場景，可依企劃調整。

> ⚠️ 本模組不內建固定劇情。每次啟動新劇集都要先走「階段 A 企劃」，再進入「階段 B 逐集製作」。

---

## 階段 A｜劇集企劃（每次新劇集啟動時做一次）

**觸發：** `@影片 劇情 企劇 [主題/方向]`

向 King 確認以下幾項（可一次列出讓 King 整批回覆，不必逐題慢問）：

1. **世界觀：** 時空背景、地點（預設＝咻咻打包倉庫，可調整）
2. **角色：** 幾位主要角色、各自身份/動機/關係
3. **主線衝突：** 一句話故事線
4. **集數規模：** 建議 8-12 集（短影音節奏，非長劇）
5. **基調：** 懸疑／愛情／職場爽劇／溫情 等

確認後輸出「劇集企劃表」：

```
# 🎬 劇集企劃表

**劇名備選：** {3個，各附一句話說明}
**世界觀：** {一句話}
**角色關係簡表：** {角色A → 關係 → 角色B...}
**分集大綱：**
第1集：{一句話} {標記}
第2集：{一句話} {標記}
...

標記：🔥關鍵集（轉折/高潮/揭秘） 💰懸念集（鉤子收尾）
```

角色確認後，依 `references/character-bible-template.md` 建立每位角色的「角色定裝聖經」（外觀錨點、服裝、識別特徵、聲音錨點），存在對話中供後續每集沿用。

**連貫性原則：** 企劃表存在對話中即可，不需另存狀態檔。但每次寫新一集前，必須主動回顧企劃表與已完成集數內容，確保角色行為、場景、道具與大綱一致；如發現前後矛盾要主動提醒 King。

## 階段 B｜逐集製作（沿用 Seedance 製作前置流程）

**觸發：** `@影片 劇情 EP[集數]`

```
【劇情EP完整前置流程】

STEP 0A｜角色三視圖
→ 列出本集出場所有真人角色（依角色定裝聖經）
→ 輸出三視圖 prompt（正面/側面/半側面）
→ 等 King 回「角色圖好了」

STEP 0B｜場景參考圖
→ 列出本集所有場景（用@代號，參照 scene-library.md）
→ 輸出每個場景3張角度的參考圖 prompt
→ 等 King 回「場景圖好了」

STEP 0C｜本集 Shot 場景預告表
→ 先列出本集所有 Shot 編號 + 場景代號 + 秒數
→ 讓 King 確認結構再出詳細 prompt

STEP 1｜Shot 腳本
→ 每個 Shot 開頭標注「場景：@代號」
→ 角色外觀完整重述，不省略
→ @image 標注對應參考圖檔名
```

**快速模式：** King 說「直接出腳本」→ 跳過等待，@image 和場景圖標「待補」繼續出。

## 鏡頭系統（通用敘事技法）

```
對白少+情緒重  → 特寫 5秒
對白多+建立關係 → 雙人鏡 15秒
聽的人比說的重要 → 過肩鏡 10秒
重要細節/道具   → 插入鏡 5秒
```

## 環境敘事原則（刺猬星球「環境敘事」技法）

```
場景本身要說故事，不只是背景：
範例：角落獨自吃飯 = 邊緣感、不屬於這裡
範例：辦公室燈還亮 = 有人還在
範例：抽屜裡的鑰匙 = 沒打開的選擇
→ 依本次企劃實際劇情，自訂類似的「場景說故事」細節，不套用固定範例

分鏡落差製造張力：
情緒高點 → 突然切到空鏡（讓觀眾消化）
平靜場景 → 突然插入緊張 insert shot
```

## 情感主題音樂錨點（依企劃自訂方法論）

```
選一個樂器/音符作為某條情感線的識別音，
隨情感推進每集遞增強度，
高潮集第一次完整呈現主題，
結尾集做最後一次呈現或淡出收尾。

→ 套用方式：依本次企劃的角色與情感線自行設計識別音與節點，
   不沿用任何固定旋律或固定集數方案。
```

## 模組規則

```
劇情式：每次新角色/新場景皆須完整建立定裝聖經與場景庫，不可省略
快速模式以外，角色與場景圖須等 King 確認才進入下一步
```
